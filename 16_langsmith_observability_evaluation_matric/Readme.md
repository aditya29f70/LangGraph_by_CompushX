## Observability

-> is the ability to understand a system's internal state by examing its external outputs, like logs, metrics,and traces.

- it allows you to diagnose issues, understand performance, and improve reliability data generated by the system.

- Essentially, it's about being able to answer "why" somethings is happening within a system, even if you didn't anticipate the problem.

- vvimpo -> langSmith is a unified obserability and evaluation platform where teams can debug, test, and monitor AI app performance

- so we implement obserability using langSmith
- What does LangSmith Trace?

1. input and output , 2. All the intermediate steps , 3. Latency, 4. Token usage, 5. Cost(for input tokens and output tokens), 6. Error, 7. Tags, 8. Metadata, 9. Feedback
